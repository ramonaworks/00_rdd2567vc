The Prompt Operating Framework

This page documents a complete AI operating framework. It converts prompt usage into structured execution. Each section contains the exact prompts, templates, and sequencing logic required to move from ad hoc usage to repeatable output generation.

Framework Components

  • Prompt Engineering Formula
  • Taste Curation System
  • Master Prompt Creation
  • Output Iteration Protocol
  • System Prompt Generation
  • AI Critic Protocol
  • Context Compression Workflow
  • Knowledge Base Structure
  • Personalized Learning Template

1. Prompt Engineering Formula

Core 4 part structure:

Role + Context + Command + Format

Base template: 

Act as [ROLE].

Context: [WHO YOU ARE + SITUATION + CONSTRAINTS + GOALS].

Task: [SPECIFIC COMMAND WITH CLEAR CRITERIA].

Format: [OUTPUT STRUCTURE: BULLETS, TABLE, PDF, WORD COUNT, STYLE RULES].

Enhancement: 

Here is an example of world class output. Match this structure and quality: [PASTE EXAMPLE].

2. Taste Curation System

Step 1. Build taste library. Collect examples of world class outputs in your field. Store phrases, structures, formats.

Step 2. Translate taste into rules. 

Max line length: 100 characters.
Write in 9th grade English.
Use similes over examples.
Keep sentences short.
Avoid cheesy quotes.
No em dashes.

Step 3. Convert into reusable taste block. 

Style rules:

- [RULE 1]
- [RULE 2]
- [RULE 3]

Always follow these rules in every response.

3. Master Prompt Creation

Step 1. Generate interview. 

Act as an interviewer. Ask me everything needed to build a master prompt for my role as [ROLE].

Step 2. Answer all questions in detail.

Step 3. Generate master prompt. 

Using all my answers above, generate a complete master prompt for my role. Make it structured and reusable.

Step 4. Generate this master prompt as a PDF document. Usage: Upload PDF into any new AI platform before prompting.

4. Output Iteration Protocol

Step 1. Upload master prompt.

Step 2. Give specific feedback.

Weak feedback: Make it punchier. 

Rewrite this.
Open with a strong reframe about [X].
Remove redundancy in paragraph 2.
Increase specificity around [Y].
Replace generic phrases with concrete metrics.

Step 3. Use canvas mode. 

Write this as a canvas document.

Lock structure: 

Use this exact format and structure for the next 10 variations.

5. System Prompt Generation

After achieving final output: 

Write the system prompt that would have generated this exact output.

Save output as PDF. Optional conversion to reusable GPT: Copy system prompt into Custom GPT instructions field.

6. AI as Critic Protocol

Act as a devil’s advocate. Stress test my assumptions. List all risks, logical flaws, blind spots, and counterarguments in detail.

Break down your criticism using first principles reasoning. Decompose each issue into fundamental truths and rebuild the argument from base assumptions.

Reintegration: 

Update my master prompt with this principle: [INSIGHT].
Rewrite the master prompt including this new constraint.

7. Context Compression Workflow

Step 1. Paste full data dump. 

Summarize this content into bullet points of key facts, data, frameworks, and stories.
Reduce to 10% of original length.

What important elements were removed that might affect accuracy or context? List them.

Lock compressed version: 

Use only the compressed summary above as context for the next task.
Start new chat. Paste compressed context.

8. Knowledge Base Organization Structure

Create project folders per initiative. 

Inside each project:

- Master prompt PDF
- Compressed context PDF
- Relevant documents
- Final system prompts

Maintain separate folder of system prompts by department. Store as PDFs for portability.

9. Personalized Learning Template

Teach me [TOPIC]. I have [X minutes].

Write a [X minute] research paper on [TOPIC] in [GRADE LEVEL] language.
Use clear explanations without unnecessary metaphors.

Audio mode: 

Generate response. Use play feature to listen.
